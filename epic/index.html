    Ambitious and in touch with new technology/fashion trends.

Goals/Needs

  • Wants to load photos on H&M website to get ideas for his fashion portfolio
  • Wants to track visitor traffic on specific clothing items
  • Wants to search for clothing and fashion trends without complication from website

Frustrations

  • Files and photos take too long to upload
  • Website freezes continually
  • The search engine on H&M is too word sensitive

User Story

As a user, Nick is an up and coming fashion designer who's is contracted by a major clothing company.
He likes to visit other clothing lines' websites to keep current in the industry.

Use Case:

Title: Nick's user page on H&M website

Description: Nick wants to gain insight by viewing changing clothing trends via H&M

Name of user and their role: Nick; fashion guru who wants to showcase his designs on the internet

Interaction Flow:

  • Nick goes to H&M.com landing page and logs into his H&M account
  • Server recognizes his account and sends him to his homepage
  • Nick clicks the Category button to search specific items
  • The category button sends request to server
  • Server searches for the data
  • The data is sent to the user
  • The user presses 'favorites' button. The button then stores the data in archive for the user
  • The user clicks to view photos, the server then sizes and loads the photo to fit the screen
  • The user presses the 'share' button
  • The share button sends to server, the server then sends shared items to various social media sites.
